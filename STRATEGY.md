# 점운 결제 + 앱스토어 전략

> 개발 방향 결정에 필요한 전략을 기록한다.
> 점운에 직접 해당하는 내용만 담는다.

---

## 현재 결제 수단 현황 (확정)

| 결제수단 | 대상 | 한도 | 상태 |
|---------|------|------|------|
| **페이앱** | 사주 결과 소액 결제 | 월 300만원 (카드만) | ✅ 승인 완료 |
| **결제선생** | 파트너 가입비 200만원+ | 고액 가능 | ✅ 연결 완료 |
| **토스** | 사주 결과 소액 결제 | 승인 후 확인 | 승인 대기 중 (1~2개월) |
| 구글 인앱결제 | 앱 내부 결제 | - | 앱 출시 후 검토 |

> 토스 승인 전까지는 **페이앱으로 소액 결제 운영 가능**.
> 토스 승인 후 병행 또는 토스 단일화 결정.

---

## 구글 인앱결제 정리

- 토스 대체제가 아님
- 웹 결제는 앱 등록 후에도 토스/페이앱 유지 가능
- 앱 안 디지털 콘텐츠 판매 시 구글이 인앱결제 요구할 수 있음 (수수료 30%)
- **결론: 웹은 기존 PG, 앱에서만 구글 인앱결제 검토**

---

## 구글플레이 등록 가능한가?

**가능.** 점운은 Next.js 웹앱 → WebView 앱 변환으로 등록 가능.

순서:
1. 구글 개발자 계정 생성 (1회 $25)
2. 웹앱 → Android 앱(AAB) 변환 ← Claude Code 작업 구간
3. 구글플레이 심사
4. 구글 인앱결제 추가 검토

---

## 앱스토어 전환 작업, 언제?

지금 바로는 아님. 선행 조건 완료 후:

1. 웹앱 나머지 기능 완성 (대운 UI 등)
2. 토스 승인 + 실제 결제 테스트
3. 실사용자 피드백 확인
4. → 앱 변환 작업 요청 (1~2주 분량)

---

## 우선순위

1. 웹앱 나머지 기능 완성
2. 페이앱으로 소액 결제 오픈
3. 토스 승인 대기 + 가상계좌 한도 확인
4. 구글플레이 앱 변환 작업
5. 구글 인앱결제 추가 검토
